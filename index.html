Frequently Asked Questions

1. the difference between var, let, and const
var is function-scoped and can be reassigned. let is block-scoped and can be reassigned but not redeclared. const is block-scoped and cannot be reassigned after initialization.
2. the difference between map(), forEach(), and filter()

map(): Transforms each element and returns a new array.

forEach(): Loops through elements but returns undefined.

filter(): Returns a new array with elements that pass a test.

3. explain arrow functions and how they are different from regular functions
Arrow functions are a shorter syntax for functions, using =>. They differ from regular functions in that they don’t have their own this or arguments, and they cannot be used as constructors with new.
4. How JavaScript Promises work
JavaScript Promises handle asynchronous operations. A promise can be pending, fulfilled, or rejected. You use .then() to handle fulfillment and .catch() to handle rejection. Promises make asynchronous code more readable and manageable.
5. how closures work in JavaScript
A closure in JavaScript is when an inner function retains access to variables from its outer function, even after the outer function has finished executing. This allows the inner function to "remember" and use those variables later.
